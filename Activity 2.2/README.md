# JS Events Practice Lab

A single-page JavaScript lab for Web 1 students. Four challenges that cover the most important event-driven patterns in frontend development.

---

## Goals

- Use `addEventListener` to respond to user interactions
- Manipulate the DOM with `classList`, `querySelector`, and `createElement`
- Work with form events and input validation
- Filter and sort data, then re-render it dynamically

---

## Setup

1. Clone or download this repo.
2. Open `index.html` in your browser — a **live server extension** (e.g. VS Code Live Server) is recommended so ES module imports work correctly.
3. All your work goes in `assets/js/script.js`. Read the TODO comments carefully.

> **Note:** The `<script type="module">` tag in `index.html` is required for the `import` statement to work. If you open the file directly (`file://`) instead of via a local server, the import will fail.

## Challenges

| # | Challenge | Key concepts |
|---|-----------|--------------|
| 1 | **Mobile Nav Toggle** | `classList.toggle`, event delegation, `contains`, `resize` |
| 2 | **Image Carousel** | index state, `transform`, prev/next logic, wrapping |

## Challenge Summaries

### 1 — Mobile Nav Toggle
Make the hamburger button in the site header open and close the nav menu on mobile. The CSS classes are already written — your job is the JavaScript logic.

**Done when:**
- [ ] Button opens and closes the menu
- [ ] Clicking outside the menu closes it
- [ ] Resizing to desktop width resets the open/closed state

---

### 2 — Image Carousel
Wire up the previous and next buttons so the carousel cycles through slides. The track, slides, and buttons exist in the HTML.

**Done when:**
- [ ] Prev / next buttons move to the correct slide
- [ ] Index wraps (last → first, first → last)
- [ ] *(Extra)* Dot indicators are generated by JS and stay in sync

---

### 3 — Username Validation
Intercept the form submit and check that the username follows the rules before accepting it.

**Rules:** no leading numbers, no spaces, no special characters (`@ # $ % & * ! ?`)

**Done when:**
- [ ] Invalid usernames show a descriptive error message
- [ ] Valid usernames show a success message
- [ ] *(Extra)* Spaces are auto-replaced with hyphens and the corrected value is shown

---

### 4 — Character Filter & Sort
Render cards from the `characters` array, then keep them updated whenever the checkboxes or sort dropdown change.

**Done when:**
- [ ] All character cards render on load
- [ ] Checking a filter attribute shows only matching characters
- [ ] The sort dropdown reorders the visible cards
- [ ] Filter and sort work together

---

## Extra Challenge — Local Fonts

Instead of loading fonts from Google Fonts, download the font files and serve them locally.

1. Go to [fonts.google.com](https://fonts.google.com) and download **Bebas Neue** and **Inter**.
2. Create an `assets/fonts/` directory and place the `.woff2` files there.
3. In `assets/css/fonts.css`, uncomment the `@font-face` blocks and remove the Google Fonts `<link>` from `index.html`.
